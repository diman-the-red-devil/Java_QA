# Условия и требования

## Условия

| №    | Описание                                                         | Примечание                                                                    |
|------|------------------------------------------------------------------|-------------------------------------------------------------------------------|
| 1.   | Предусловия                                                      |                                                                               |
| 1.1. | **Выполнены все условия, исправлены все недочеты предыдущих ДЗ** | Каждый **неисправленный недочет** будет считаться как **5 допущенных ошибок** |
| 1.2. | **Сохранены все наработки предыдущих ДЗ**                        |                                                                               |
| *    | Параметризованный запуск через **-D** параметры                  |                                                                               |
| *    | Паттерн **WebDriverFactory** и настройка драйвера                |                                                                               |
| *    | Явные и где нужно неявные ожидания                               |                                                                               |
| *    | Применение **Action**                                            |                                                                               |
| *    | Работа с окнами                                                  |                                                                               |
| *    | Исполнение скриптов **JS** где необходимо                        |                                                                               |
| *    | Протоколирование через события драйвера                          |                                                                               |
| *    | Скриншотирование и логирование                                   |                                                                               |
| *    | Паттерн **Page Object Model + Page Factory**                     |                                                                               |
| *    | Паттерн **Arrange Act Assert**                                   |                                                                               |
| *    | Паттерн **Assert Object / Matchers**                             |                                                                               |
| *    | Паттерн **Page Elements**                                        |                                                                               |
| 2.   | Переписать автотесты с применением подхода **BDD**:              |                                                                               |
| 2.1. | Каждый кейс как отдельный **feature** файл                       |                                                                               |
| 2.2. | Каждый кейс разбить на сценарии                                  | Один сценарий - одна проверка (см пример ниже)                                |
| 3.   | Вынести общие шаги сценариев в **Предысторию**                   | Один **feature** файл - одна предыстория                                      |
| 4.   | Сделать шаги и проверки параметризованными                       |                                                                               |
| 5.   | Добавить **Структуру сценария**                                  | В проверках будут проверяться различные варианты результата (больше 2-х)      |
| 6.   | Добавить таблицы в качестве параметров                           | В любые шаги и проверки                                                       |
| 7.   | Добавить теги сценариям                                          | Названия на ваше усмотрение                                                   |
| 8.   | Добавить конфиг **cucumber.properties**                          | Теги, плагины и т. д.                                                         |
